# 논문 해설 자료 제작 지침

이것은 ML/CV 논문을 한국어 "해설 자료"로 만드는 작업의 지침입니다. 사용자는 사진 참조를 위해 해설할 **논문 PDF**를 제공합니다.

## 무엇을 만드는가

- 원문의 복잡한 개념을 이해하기 쉽게 읽는 해설 자료입니다. **단순 번역이 아니라**, 핵심 아이디어를 풀어서 설명하고 그 논문이 연구 흐름에서 차지하는 위치를 보여주는 글입니다.
- 독자는 선행 연구를 어느 정도 알고 있다고 가정합니다. 그러므로 일반론보다 **그 논문 고유의 기여와 직관**에 집중합니다.
- 한국어, **"-입니다" 체**. 표준 영어 기술 용어(attention, inductive bias, patch 등)는 영어로 유지하되 처음 나올 때 풀어 설명합니다.

## 반드시 담아야 할 내용 (구조는 논문에 맞게 자유롭게)

아래는 **포함할 내용**이지 고정된 섹션 순서가 아닙니다. 논문에 가장 잘 맞는 흐름으로 자유롭게 구성하세요.

- 이 논문의 핵심 기여를 압축해 전달.
- **논문이 도입한 핵심 메커니즘/아이디어**: 각각 관련 수식 + 직관 + 도판과 함께.
- **어려운 개념의 직관**: 비유·예시·그림으로 와닿게. 수식을 그대로 다시 적기만 하지 말 것.
- 이 논문이 무엇 위에 섰고(선행), 무엇을 바꿨으며, 무엇으로 이어졌는지(후속)를 본문 전체에 녹이고, 직접 만든 계보도로 시각화합니다.
- **중요한 실험 결과**가 있을 경우, 핵심 수치를 짚되 **원문 표를 그대로 옮기지 말고 다시 정리**. 모델 변형·하이퍼파라미터 같은 참고 표가 도움되면 추가.
- 다른 논문과의 교차점이 있으면 **자연스럽게 교차 참조**(개념의 출처·후속을 연결).

제목은 [논문 리뷰]<논문 제목>(<메인 아이디어>).
제목과 인용(저자·학회·연도·arXiv) 다음에는 **곧바로 본문으로** 들어갑니다. "이 자료에 대하여 / 목적 / 독자 가정 / 표기 매핑 / 도판 출처" 같은 도입 안내는 **넣지 않습니다**.

## 형식·산출 규칙

**산출물 3종** (파일명은 `YYYY-MM-DD-<main-idea>` 로 통일: 날짜=논문을 읽은 날, `<main-idea>`=논문 핵심을 나타내는 짧은 소문자 슬러그 예 `transformer`/`vit`/`swin`):
- `YYYY-MM-DD-<main-idea>.md` — 블로그용
- `YYYY-MM-DD-<main-idea>.pdf` — 인쇄·열람용(자체 완결, 이미지 임베드)
- `YYYY-MM-DD-<main-idea>-images.zip` — 이미지 묶음

**YAML 머리말**을 .md 최상단에 둡니다:

```
---
layout: post
title: "<논문 제목>"
date: <읽은 날짜, YYYY-MM-DD>
authors: "<예: Liu et al.>"
venue: "<예: ICCV>"
year: <연도>
tags: [<주제 키워드들>]
---
```

**수식**은 **실제 LaTeX**로 작성합니다 — 인라인 `$...$`, 디스플레이 `$$...$$`. 수식을 이미지로 만들지 않습니다. (블로그는 MathJax/KaTeX로, PDF는 아래 빌드의 `pandoc --mathml`로 렌더)

**그림은 두 종류**:
- **직접 만든 그림**(계보도 등): matplotlib, **투명 배경**(`savefig(..., transparent=True)`).
- **논문 도판**: PDF에서 잘라옴, **흰 배경 유지**.

**이미지 경로**: .md에서 `/assets/img/<main-idea>/<파일명>.png` (Jekyll 자산 절대경로). zip은 `assets/img/<main-idea>/…` 구조로 묶어 블로그 **사이트 루트**에 풀면 맞습니다.

## 기술 파이프라인 (검증된 레시피)

오프라인 환경(Noto CJK 폰트 설치, pandoc·poppler·Playwright Chromium·matplotlib·Pillow 사용 가능). 작업 전 관련 SKILL.md(pdf, pdf-reading 등)를 먼저 확인합니다.

### 논문 도판 추출

페이지를 렌더링한 뒤 잘라내는 방식이 깔끔합니다. PNG는 블로그 경로 구조와 같은 `assets/img/<main-idea>/`에 저장합니다.

```bash
pdftoppm -png -r 200 -f <페이지> -l <페이지> "<논문.pdf>" raster/page   # letter 기준 1700x2200 px
```
```python
from PIL import Image, ImageOps
import os
IMGDIR = "assets/img/<main-idea>"; os.makedirs(IMGDIR, exist_ok=True)
def crop(page, box, out, pad=10):                 # box=(left,top,right,bottom)
    im = Image.open(f"raster/page-{page:02d}.png").convert("RGB")
    c = ImageOps.expand(im.crop(box), border=pad, fill="white")
    c.save(f"{IMGDIR}/{out}"); print(out, c.size)
```
- 잘라낸 결과를 **반드시 `view`로 확인**하고, 제목·축·범례가 잘렸으면 좌표를 넓혀 다시 자릅니다. 캡션은 제외.

### 계보도 (matplotlib, 투명 배경)

3열(선행 ↔ 본 논문 ↔ 후속)에 둥근 박스 + 화살표. 한글은 Noto CJK 폰트를 파일 경로로 등록.

### PDF 빌드 (머리말 제거 → 이미지 경로 보정 → pandoc MathML → Chromium)

```python
# (a) YAML 머리말 제거 + 이미지 절대경로를 상대경로로 (PDF용)
import re
src = open("YYYY-MM-DD-<main-idea>.md", encoding="utf-8").read()
body = re.sub(r'^---\n.*?\n---\n', '', src, count=1, flags=re.DOTALL)   # 머리말 제거
body = body.replace("/assets/img/", "assets/img/")                     # 절대→상대 (file:// 해석용)
open("_nofm.md","w",encoding="utf-8").write(body)
# 실제 PNG는 작업폴더의 assets/img/<main-idea>/ 에, _pdf.html 도 작업폴더 루트에 둘 것.
```
```bash
# (b) GFM(파이프 표·raw HTML) + 수식 MathML (오프라인에서 Chromium이 네이티브 렌더)
pandoc _nofm.md --from gfm --to html5 --mathml -o _body.html
```

## 마무리 점검

- 잘라낸 도판을 `view`로 확인(잘림 없는지).
- PDF 빌드 후 대표 페이지 몇 장을 rasterize해 **수식·표·그림·페이지 나눔·한글 폰트**를 점검.
- .md의 모든 `/assets/img/<main-idea>/…png` 참조가 실제 파일과 일치하는지 검증(누락 0).
