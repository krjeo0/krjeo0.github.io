# Swin Transformer 정밀 독해 — *Hierarchical Vision Transformer using Shifted Windows* 함께 읽기

> Liu et al., *Swin Transformer: Hierarchical Vision Transformer using Shifted Windows*, **ICCV 2021** (arXiv:2103.14030) 해설 자료입니다.

---

## 이 자료에 대하여

이 글은 Swin Transformer 논문을 **나란히 펴 놓고 읽기 위한 해설**입니다. 번역본이 아니라, 이 논문이 **새로 도입한 핵심 메커니즘**(윈도우 기반 attention, shifted window, patch merging, relative position bias 등)과 그 직관, 그리고 비전 백본 설계 흐름에서의 위치를 짚는 것이 목적입니다.

- **독자 가정**: Transformer/self-attention과 **ViT**의 기본 구조는 안다고 가정합니다(앞서 만든 ViT 해설을 읽었다면 곳곳의 연결이 더 잘 보입니다). 따라서 attention 일반론보다 **Swin이 ViT의 무엇을 어떻게 바꿨는지**에 집중합니다.
- **"notation"의 의미**: 여기서 짚는 notation은 H, W 같은 자명한 기호가 아니라, **새로운 아이디어가 담긴 개념적 장치**(W-MSA, SW-MSA, B 등)를 말합니다. 자명한 기호는 본문 흐름에서만 풀고, 부록 A에 핵심 메커니즘을 따로 정리했습니다.
- **표기 매핑**: 본문의 수식·도판은 논문 원본 번호(예: 논문 Eq. (3), 논문 Figure 2, 논문 Table 4, 논문 §3.2)와 연결해 두었습니다.
- **도판 출처**: 본문에 삽입된 아키텍처·개념 그림은 논문 원본 도판이고, 계보도와 수식 박스는 이 해설을 위해 새로 제작했습니다.

## 0. 한 문장 요약

> **ViT는 분류엔 강하지만 (a) 토큰 스케일이 고정이고 (b) 이미지 크기에 제곱으로 비싸져 검출·분할 같은 밀집 예측의 범용 백본으로는 부적합하다. Swin은 "계층적 feature map + 윈도우 기반(특히 shifted window) self-attention"으로 이 둘을 동시에 해결해, 분류를 넘어 검출·분할까지 아우르는 범용 비전 백본이 된다.**

ViT 해설의 슬로건이 *"데이터 규모 > inductive bias"* 였다면, Swin의 메시지는 그 반대 방향에서 흥미롭습니다 — **"범용 비전, 특히 밀집 예측에는 CNN이 갖던 inductive bias(locality·계층 구조)가 여전히 유익하다"**. Swin은 그 inductive bias를 Transformer에 되살립니다.

## 1. 계보적 위치 — Swin은 어디에 서 있는가

  <img src="/assets/img/swin/fig0_genealogy.png" alt="Swin Transformer의 계보적 위치 다이어그램" style="width:100%;">

</figure>

**선행 연구.** 직접 부모는 **ViT (2020)** 입니다. ViT는 패치를 토큰으로 보고 순수 Transformer를 적용해 분류에서 인상적인 속도-정확도를 보였지만, **단일 저해상도 feature map**과 **이미지 크기에 제곱인 복잡도** 때문에 밀집 예측엔 부적합했습니다. **DeiT (2021)** 는 ViT를 ImageNet-1K만으로 학습 가능하게 만들었고, **CNN 백본 + FPN/U-Net** 계열은 *계층적 피라미드*라는 설계 철학을 제공했습니다. 한편 **local self-attention (2019)** 류는 픽셀마다 윈도우를 두는 *sliding window* 방식이라 모델링은 되지만 **하드웨어 메모리 접근이 비효율적**이었고, 동시기의 **PVT (2021)** 는 멀티스케일 feature map을 시도했으나 **여전히 quadratic** 복잡도였습니다.

**Swin이 던진 질문.** "Transformer를 분류뿐 아니라 **검출·분할까지 아우르는 범용 백본**으로 만들 수 있는가? ViT의 두 약점(고정 스케일, quadratic 복잡도)을 어떻게 동시에 풀 것인가?"

**이후로 이어진 것.** **Swin V2** 가 스케일업·고해상도 안정화를 이어갔고, **ConvNeXt (2022)** 는 "Swin이 그렇게 잘된다면, CNN을 제대로 현대화하면 대등하지 않나?"라며 **CNN으로 맞대결**했습니다(중요한 반론). **ViTDet (2022)** 은 "plain ViT도 검출 백본으로 충분하다"며 **계층 구조가 필수는 아님**을 보였습니다(또 다른 반론). 그럼에도 *hierarchical + window* 설계는 **MViT/PVTv2** 등으로 이어지며 비전 백본 설계의 한 표준 축이 되었고, 검출·분할 응용(예: SAM)의 백본으로 폭넓게 채택되었습니다.

> **ViT 해설과의 연결.** ViT 자료에서 *"ViT가 버린 inductive bias를 후속 연구가 부분적으로 회수하는 변증법"* 이라고 예고했는데, **Swin이 바로 그 대표 사례**입니다. ViT가 "다 버려도 데이터로 배운다"였다면, Swin은 "범용·밀집 예측엔 locality·계층을 도로 넣는 게 낫다"입니다.

## 2. 왜 ViT로는 안 되는가 — 두 가지 modality 차이 (논문 Figure 1)

Swin의 출발점은 **언어와 비전의 두 가지 차이**입니다.

1. **스케일 변동(scale variation)**: 문장의 단어 토큰은 스케일이 일정하지만, 이미지 속 시각 객체는 **크기가 천차만별**입니다(작은 새 vs 화면을 꽉 채운 건물). ViT처럼 **토큰 스케일을 하나로 고정**하면 이 다양성을 담기 어렵습니다 — 특히 객체 검출처럼 스케일이 핵심인 과제에서요.
2. **높은 해상도(픽셀 수)**: 픽셀은 단어보다 압도적으로 많습니다. 의미 분할(semantic segmentation)처럼 **픽셀 단위 밀집 예측**이 필요한 과제에서, **이미지 크기에 제곱인** self-attention은 감당이 안 됩니다.

<figure style="max-width:720px; margin:1.6em auto; text-align:center;">
  <img src="/assets/img/swin/fig1_hierarchy_vs_vit.png" alt="Swin 계층적 feature map vs ViT 단일 해상도 비교" style="width:100%;">
  <figcaption style="font-size:0.9em; color:#555; margin-top:0.5em;"><b>논문 Figure 1.</b> (a) Swin은 깊어질수록 패치를 병합해 <b>계층적 feature map</b>(4×→8×→16×)을 만들고, self-attention을 <b>각 국소 윈도우(빨강) 안에서만</b> 계산해 이미지 크기에 <b>선형</b> 복잡도를 가집니다. (b) 기존 ViT는 <b>단일 저해상도</b> feature map만 만들고 전역 attention이라 <b>제곱</b> 복잡도입니다.</figcaption>
</figure>

핵심 대비는 이렇습니다. ViT는 처음부터 끝까지 **하나의 낮은 해상도**(예: 16× 다운샘플)만 내놓습니다 — 분류엔 충분하지만, 검출·분할에 쓰이는 **FPN(feature pyramid network)이나 U-Net** 같은 도구는 *여러 해상도의 feature map*을 전제로 하므로 ViT와는 잘 맞지 않습니다. Swin은 **CNN처럼 4×/8×/16×/32× 피라미드**를 만들어 이 도구들을 그대로 꽂을 수 있게 합니다. 이것이 "범용 백본"의 실체입니다.

## 3. 핵심 메커니즘 ① — 계층적 구조 (Hierarchical Feature Maps)

<figure style="max-width:820px; margin:1.6em auto; text-align:center;">
  <img src="/assets/img/swin/fig3_architecture.png" alt="Swin Transformer 전체 아키텍처와 두 블록" style="width:100%;">
  <figcaption style="font-size:0.9em; color:#555; margin-top:0.5em;"><b>논문 Figure 3.</b> (a) Swin-T의 전체 구조: Patch Partition → Linear Embedding → 4개의 Stage(사이사이 Patch Merging). 해상도는 H/4→H/8→H/16→H/32, 채널은 C→2C→4C→8C로 변합니다. (b) 연속한 두 Swin 블록: <b>W-MSA</b> 블록과 <b>SW-MSA</b> 블록이 한 쌍입니다.</figcaption>
</figure>

### 3.1 시작: Patch Partition + Linear Embedding

입력 RGB 이미지를 **겹치지 않는 4×4 패치**로 자릅니다(ViT의 16×16보다 **훨씬 잘게** — 이것이 고해상도 feature map의 출발점). 각 패치의 raw feature는 4×4×3 = 48차원이고, **선형 임베딩**으로 임의 차원 C에 사영합니다. 여기까지가 토큰화입니다(개념적으로 ViT의 patchify와 같지만 패치가 작습니다).

### 3.2 새 개념: Patch Merging (계층을 만드는 다운샘플링)

깊어지면서 토큰 수를 줄이고 채널을 키우는 장치가 **Patch Merging**입니다. **2×2 이웃 토큰의 feature를 concat**(→ 4C차원)한 뒤 **선형층으로 2C로** 사영합니다.

<div style="text-align:center; margin:1.1em 0;"><img src="/assets/img/swin/eq_merge.png" alt="patch merging: 2x2 이웃 concat 4C에서 선형사영 2C" style="max-width:520px; width:88%;"></div>

결과적으로 **토큰 수는 4배 줄고(해상도 2배 다운샘플), 채널은 2배** 늘어납니다. 이는 CNN에서 **stride-2 pooling/conv가 하던 다운샘플링**과 정확히 같은 역할입니다. CNN의 직관을 Transformer로 옮긴 셈입니다.

### 3.3 Stage 1~4: ResNet과 동형인 피라미드

이 과정을 반복해 **4개의 Stage**를 만듭니다. 출력 해상도는 차례로 **H/4, H/8, H/16, H/32**(즉 4×→8×→16×→32× 다운샘플), 채널은 **C→2C→4C→8C**입니다. 이는 VGG·ResNet의 feature map 해상도 단계와 **동일**합니다. 그래서 Swin은 **기존 비전 프레임워크의 백본을 그대로 교체**할 수 있습니다 — 이것이 "general-purpose backbone"의 기술적 근거입니다.

> **Swin-T 기준** 각 Stage의 블록 수는 {2, 2, 6, 2}입니다. 블록 수가 모두 **짝수**인 데는 이유가 있습니다(§5에서 설명).

## 4. 핵심 메커니즘 ② — 윈도우 기반 self-attention과 선형 복잡도

ViT는 모든 토큰이 다른 모든 토큰에 attend하는 **전역(global) self-attention**을 씁니다. 토큰 수에 **제곱**이라, 밀집 예측이나 고해상도에선 토큰이 너무 많아 감당이 안 됩니다.

### 4.1 새 개념: W-MSA (Window-based Multi-head Self-Attention)

Swin은 self-attention을 **겹치지 않는 M×M 윈도우 안에서만** 계산합니다(기본 M=7). 이미지를 윈도우로 균등 분할하고, **각 윈도우 내부에서만** 토큰들이 서로 attend합니다.

복잡도를 보면 이 설계의 핵심이 드러납니다. h×w 패치 이미지에서:

<div style="text-align:center; margin:1.1em 0;"><img src="/assets/img/swin/eq_complexity.png" alt="MSA와 W-MSA의 복잡도 비교 수식" style="max-width:560px; width:90%;"></div>

(논문 Eq. 1, 2) 위쪽 전역 MSA는 토큰 수 hw에 대해 **제곱**(2(hw)²C 항)인 반면, 아래쪽 W-MSA는 윈도우 크기 M을 고정하면 hw에 대해 **선형**(2M²hwC 항)입니다.

**직관**: 이미지가 커져도 윈도우 한 칸의 크기(M×M)는 그대로이므로, **윈도우 개수만 픽셀 수에 비례해 늘어납니다.** 윈도우당 attention 비용은 일정하니, 총비용이 픽셀 수에 **선형**이 됩니다. 여기서 **M은 locality를 조절하는 손잡이**이자 linear 복잡도의 근거입니다 — ViT 해설에서 본 "패치가 작을수록 attention이 제곱으로 비싸진다"는 문제를, Swin은 *attention 범위 자체를 윈도우로 제한*해서 푼 것입니다.

## 5. 핵심 메커니즘 ③ — Shifted Window (이 논문의 심장)

W-MSA에는 치명적 약점이 있습니다. **윈도우 안에서만 attend하므로 윈도우 사이에 정보 교류가 전혀 없습니다.** 이러면 표현력이 제한됩니다(전역 맥락을 못 봄). Swin의 이름이 된 해결책이 **Shifted Window**입니다.

### 5.1 새 개념: SW-MSA (Shifted Window MSA)

연속한 두 블록에서 **윈도우 분할 자체를 이동(shift)** 시킵니다. 한 블록은 정규 분할로 attend하고(**W-MSA**), 바로 다음 블록은 윈도우를 **(⌊M/2⌋, ⌊M/2⌋)만큼 대각선으로 밀어** 새 윈도우로 attend합니다(**SW-MSA**).

<div style="text-align:center; margin:1.0em 0;"><img src="/assets/img/swin/eq_shift.png" alt="윈도우 이동량 (floor(M/2), floor(M/2))" style="max-width:240px; width:55%;"></div>

<figure style="max-width:720px; margin:1.5em auto; text-align:center;">
  <img src="/assets/img/swin/fig2_shifted_window.png" alt="shifted window 개념: Layer l 정규 분할에서 Layer l+1 shifted 분할로" style="width:100%;">
  <figcaption style="font-size:0.9em; color:#555; margin-top:0.5em;"><b>논문 Figure 2.</b> Layer <i>l</i>(왼쪽)은 정규 윈도우 분할로 각 윈도우 안에서 attention을 계산합니다. Layer <i>l+1</i>(오른쪽)은 분할을 이동시켜 새 윈도우를 만드는데, 이 새 윈도우들이 <b>이전 층 윈도우들의 경계를 가로질러</b> 윈도우 간 연결을 제공합니다.</figcaption>
</figure>

핵심은 **shifted 윈도우가 이전 층 윈도우들의 경계를 가로지른다**는 점입니다. 그래서 SW-MSA를 거치면 **이전에 다른 윈도우에 갇혀 있던 토큰들이 서로 정보를 주고받게** 됩니다. 효율(윈도우 단위 계산)은 유지하면서 **cross-window connection**을 얻는 것입니다.

### 5.2 W-MSA 블록과 SW-MSA 블록은 한 쌍 (논문 Eq. 3)

그래서 Swin 블록은 항상 **두 개가 한 쌍**으로 동작합니다 — W-MSA 블록 다음에 SW-MSA 블록.

<div style="text-align:center; margin:1.0em 0;"><img src="/assets/img/swin/eq_block.png" alt="연속한 두 Swin 블록의 수식: W-MSA 블록과 SW-MSA 블록" style="max-width:560px; width:92%;"></div>

각 블록 구조는 ViT와 닮았습니다 — **LN을 앞에 두고(Pre-LN)**, (S)W-MSA → MLP(GELU 2층) 순서로, 각 모듈 뒤에 residual을 더합니다. 차이는 **MSA가 (S)W-MSA로 바뀐 것**뿐입니다. 여기서 ẑ는 (S)W-MSA 모듈의 출력, z는 MLP 모듈의 출력입니다.

> **헷갈리는 포인트 해소.** Swin 모델들의 블록 수가 {2, 2, 18, 2}처럼 **모두 짝수**인 이유가 바로 이것입니다 — **W-MSA + SW-MSA가 한 쌍**이라 짝수로 떨어집니다.

논문 Table 4의 ablation은 shifted window의 효과를 분명히 보여줍니다 — 단일 분할만 쓸 때 대비 **ImageNet +1.1%, COCO +2.8 box AP, ADE20K +2.8 mIoU**. 윈도우를 잇는 것이 분류뿐 아니라 밀집 예측에서 특히 중요함을 알 수 있습니다.

### 5.3 왜 sliding window가 아니라 shifted window인가

"윈도우 간 연결"이라면 픽셀마다 윈도우를 미끄러뜨리는 **sliding window**(local self-attention 류)도 떠오릅니다. 하지만 sliding window는 **query 픽셀마다 key 집합이 달라** 하드웨어에서 메모리 접근이 비효율적입니다. 반면 shifted window는 **한 윈도우 안의 모든 query가 같은 key 집합을 공유**하므로 훨씬 하드웨어 친화적입니다. 논문 Table 5·6은 shifted window가 sliding window보다 **훨씬 빠르면서 정확도는 대등**함을 보입니다.

## 6. 핵심 메커니즘 ④ — Cyclic Shift + Masking (효율적 배치 계산)

Shifted window에는 실무적 골칫거리가 하나 있습니다. 윈도우를 밀면 **윈도우 개수가 늘고**(예: 2×2 → 3×3, 약 2.25배), **가장자리 윈도우는 M×M보다 작아집니다.** 순진하게 작은 윈도우를 M×M로 패딩하고 마스킹하면 계산량이 늘어 손해입니다.

<figure style="max-width:720px; margin:1.5em auto; text-align:center;">
  <img src="/assets/img/swin/fig4_cyclic_shift.png" alt="cyclic shift와 masked MSA를 이용한 효율적 배치 계산" style="width:100%;">
  <figcaption style="font-size:0.9em; color:#555; margin-top:0.5em;"><b>논문 Figure 4.</b> feature map을 좌상단으로 <b>순환 이동(cyclic shift)</b> 시킨 뒤, 인접하지 않게 섞인 sub-window끼리는 attend하지 못하도록 <b>마스킹</b>하고(masked MSA), 끝나면 <b>역방향으로 되돌립니다(reverse cyclic shift)</b>.</figcaption>
</figure>

Swin의 해결책은 **cyclic shift**입니다. feature map 전체를 좌상단 방향으로 순환 이동시키면, 떨어져 있던 가장자리 조각들이 한 윈도우 안으로 모입니다. 단, 이렇게 모인 조각들은 원래 공간상 **인접하지 않으므로** 서로 attend하면 안 됩니다 — 이를 **마스킹**으로 막습니다(같은 sub-window 안에서만 attention 허용). 계산이 끝나면 역방향 순환 이동으로 원위치시킵니다.

이 트릭의 핵심 이득: **윈도우 개수가 정규 분할과 똑같이 유지**되므로 패딩 방식의 추가 비용이 사라집니다. 구현 디테일처럼 보이지만, Swin이 "선형 + 실제로도 빠름"을 달성하는 데 결정적인 부분입니다(논문 Table 5에서 cyclic 구현이 padding 대비 13~18% 빠름).

## 7. 핵심 메커니즘 ⑤ — Relative Position Bias (또 하나의 중요한 notation)

위치 정보를 넣는 방식에서도 Swin은 ViT와 갈립니다. ViT는 입력에 **절대(absolute) 위치 임베딩**을 더했지만, Swin은 attention을 계산할 때 **상대 위치 바이어스 B**를 logit에 직접 더합니다.

<div style="text-align:center; margin:1.1em 0;"><img src="/assets/img/swin/eq_attention.png" alt="relative position bias B를 더한 attention 수식" style="max-width:600px; width:92%;"></div>

(논문 Eq. 4) 여기서 Q, K, V는 한 윈도우 안의 query/key/value 행렬(각 M²×d), d는 query/key 차원, **B는 각 head마다 더해지는 상대 위치 바이어스**입니다. 윈도우 안에서 두 패치의 상대 위치가 각 축마다 [−M+1, M−1] 범위이므로, 실제로는 **작은 바이어스 표 B̂를 파라미터화**하고 B의 값을 거기서 인덱싱해 가져옵니다.

<div style="text-align:center; margin:1.0em 0;"><img src="/assets/img/swin/eq_relbias.png" alt="B와 B-hat의 차원" style="max-width:520px; width:88%;"></div>

**왜 상대 위치인가 (그리고 ViT 해설과의 연결).** ViT 해설 §7.2에서 *"위치 인코딩은 이후 계속 재설계된다 — Swin의 상대 위치 바이어스, 2D-RoPE 등"* 이라고 예고했는데, **여기가 바로 그 장면**입니다. 논문 Table 4는, 상대 위치 바이어스가 위치 인코딩 없음/절대 위치 임베딩보다 **분류·검출·분할 모두에서 더 좋고**, 오히려 **절대 위치 임베딩을 추가하면 밀집 예측 성능이 떨어진다**고 보고합니다. 상대 위치 바이어스는 일종의 **translation invariance라는 inductive bias**를 부드럽게 다시 넣는 장치인 셈입니다. 사전학습한 B는 다른 윈도우 크기로 **bi-cubic 보간**해 파인튜닝 초기화에 쓸 수 있습니다(ViT의 위치 임베딩 보간과 같은 발상).

## 8. 실험 하이라이트 — "범용 백본"의 증명

Swin은 세 과제에서 강합니다. 핵심은 **하나의 백본으로 분류·검출·분할을 모두** 잘한다는 점입니다.

**ImageNet-1K 분류 (논문 Table 1).** Swin은 비슷한 복잡도의 DeiT를 분명히 앞섭니다.

| 모델 | 입력 | #param | FLOPs | ImageNet top-1 |
|---|---|---|---|---|
| DeiT-S | 224² | 22M | 4.6G | 79.8 |
| Swin-T | 224² | 29M | 4.5G | **81.3** |
| DeiT-B | 384² | 86M | 55.4G | 83.1 |
| Swin-B | 384² | 88M | 47.0G | **84.5** |
| Swin-B (22K 사전학습) | 384² | 88M | 47.0G | 86.4 |
| Swin-L (22K 사전학습) | 384² | 197M | 103.9G | **87.3** |

**COCO 검출·분할 (논문 Table 2).** 동일 프레임워크에서 백본만 바꿔도 ResNe(X)t·DeiT 대비 큰 폭으로 향상하며, 최고 모델은 **58.7 box AP / 51.1 mask AP** 로 당시 SOTA를 +2.7 box AP, +2.6 mask AP 갱신했습니다.

**ADE20K 분할 (논문 Table 3).** Swin-L(22K 사전학습)이 **53.5 mIoU** 로 이전 최고(SETR, 50.3) 대비 +3.2 mIoU.

> **진짜 강점은 "꽂을 수 있다"는 것.** Swin과 ResNe(X)t는 계층적 feature map을 내므로 Cascade Mask R-CNN, FPN, UperNet 같은 프레임워크에 **백본만 갈아끼우면** 됩니다. 반면 DeiT는 단일 해상도라 deconvolution으로 **억지로 피라미드를 만들어야** 비교가 가능했고, 그마저 quadratic 복잡도 탓에 느렸습니다(예: Swin-T 15.3 FPS vs DeiT-S 10.4 FPS). 이 "범용성"이 ViT 대비 Swin의 핵심 차별점입니다.

## 9. Ablation에서 배우는 것 (논문이 강조하는 인사이트)

- **Shifted window가 핵심이다 (Table 4).** 단일 분할 대비 세 과제 모두에서 향상, 특히 밀집 예측에서 크게(+2.8 box AP / +2.8 mIoU). 지연(latency) 오버헤드는 작습니다(Table 5).
- **상대 위치 바이어스 > 절대 위치 임베딩 (Table 4).** 특히 검출·분할에서 그렇고, 절대 위치 임베딩을 더하면 오히려 밀집 예측이 나빠집니다.
- **Cyclic 구현이 빠르다 (Table 5).** padding 방식 대비 13~18% 속도 향상. Performer 같은 효율 attention보다도 비슷하게 빠르면서 **정확도는 +2.3%** 높습니다(Table 6).
- **가장 중요한 사상적 포인트.** 논문은 명시적으로 말합니다 — *"ViT/DeiT는 분류에서 translation invariance를 버렸지만, 우리는 일정한 translation invariance를 장려하는 inductive bias가 범용 비전, 특히 밀집 예측에는 여전히 유익하다고 본다."* 이는 ViT 해설의 *"데이터 규모 > inductive bias"* 에 대한 **균형추**로 읽으면 계보가 선명해집니다. *분류에서는* 큰 데이터가 inductive bias를 대체할 수 있어도, *검출·분할 같은 밀집 예측에서는* locality·계층 같은 inductive bias가 여전히 값지다는 것입니다.

## 10. 자주 헷갈리는 포인트 / 핵심 takeaway

- **W-MSA는 윈도우 안에서만 global, 윈도우 사이는 단절** → SW-MSA가 그 단절을 잇습니다. 둘은 **한 쌍**입니다.
- **블록 수가 모두 짝수**({2,2,6,2} 등)인 이유 = W-MSA + SW-MSA 쌍.
- **"linear 복잡도"는 윈도우 크기 M을 고정**하기 때문입니다. M은 locality를 정하는 손잡이입니다.
- **분류 head는 CLS 토큰을 안 쓰고 GAP**(global average pooling)를 씁니다(논문 부록 A2.1). ViT 해설 §3.1에서 말한 *"이후 백본은 CLS 대신 GAP"* 흐름 그대로입니다.
- **Swin은 4×4 패치로 시작**(ViT 16×16보다 고해상도)하고 **patch merging으로 계층**을 만들기에 밀집 예측이 가능합니다.
- **상대 위치 바이어스가 절대 위치보다 낫습니다**(특히 밀집 예측). 절대 위치 추가는 오히려 해롭습니다.
- **cyclic shift + masking**은 "선형이면서 실제로도 빠름"을 만드는 구현 트릭입니다.

## 11. 이 논문이 남긴 것 — 계보 후행 한눈에

<figure style="max-width:780px; margin:1.4em auto; text-align:center;">
  <img src="/assets/img/swin/fig0_genealogy.png" alt="Swin 계보도 (재게시)" style="width:100%;">
</figure>

- **Swin V2 (2021–22)** — 고해상도·대규모로 안정적으로 스케일업하는 기법(post-norm, scaled cosine attention 등)을 더했습니다.
- **ConvNeXt (2022)** — "계층적 Transformer가 그렇게 잘된다면, ResNet을 현대적 레시피로 다시 설계하면?"이라는 문제의식으로 **순수 CNN으로 Swin과 대등한 성능**을 보여, *Transformer가 본질인가 vs 학습 레시피·매크로 설계가 본질인가* 논쟁을 촉발했습니다(중요한 반론).
- **ViTDet (2022)** — **plain ViT**에 간단한 feature pyramid를 얹어 검출을 잘 수행, **계층적 백본이 필수는 아님**을 보였습니다(또 다른 반론).
- **MViT / PVTv2** 등 멀티스케일 Transformer 계열로 이어졌고, **검출·분할 응용(예: SAM)** 의 백본으로 폭넓게 채택되었습니다.
- 부록의 **Swin-Mixer** 실험은, hierarchical + shifted window 설계가 **MLP 계열(MLP-Mixer)에도 일반화**됨을 보입니다 — 설계 아이디어가 attention에 국한되지 않는다는 증거입니다.

> **한 줄 정리.** Swin은 "Transformer를 분류를 넘어 **검출·분할까지 아우르는 범용 백본**으로" 만든 결정적 작업이자, ViT가 버린 **CNN적 inductive bias(locality·계층)를 Transformer에 되살린** 대표 사례입니다. ViT 해설과 함께 읽으면, 2020~2022년 비전 백본 설계가 *"귀납 편향을 얼마나 버리고 얼마나 되살릴 것인가"* 라는 한 축을 두고 진동한 흐름이 보입니다.

## 부록 A. 핵심 개념(notation) 정리 — 새 메커니즘 중심

> 자명한 기호(h, w, C 등)가 아니라, 이 논문이 **새로 도입한 개념적 장치**만 모았습니다.

| 개념 (notation) | 의미와 역할 |
|---|---|
| **W-MSA** | 겹치지 않는 M×M 윈도우 **내부에서만** 수행하는 multi-head self-attention. 전역 대신 국소 → 비용을 이미지 크기에 **선형**으로. |
| **SW-MSA** | 윈도우 분할을 (⌊M/2⌋, ⌊M/2⌋)만큼 **이동**시킨 뒤 수행하는 윈도우 attention. 이전 층 윈도우 경계를 가로질러 **윈도우 간 연결**을 만듦. |
| **M (window size)** | 윈도우 한 변의 패치 수(기본 7). **locality를 조절하는 손잡이**이자 W-MSA가 선형 복잡도를 갖는 근거. |
| **Patch Merging** | 2×2 이웃 토큰을 concat(→4C) 후 선형사영(→2C). 토큰 4배↓·채널 2배↑. CNN의 **stride-2 다운샘플링**에 대응. |
| **Stage / 계층 구조** | 해상도 4×→8×→16×→32×, 채널 C→2C→4C→8C의 피라미드. ResNet과 **동형** → FPN·UperNet 등에 그대로 호환. |
| **B (relative position bias)** | attention logit에 더하는 **상대 위치 바이어스**(head마다). 작은 표 B̂ ∈ ℝ^((2M−1)×(2M−1))에서 인덱싱. ViT의 절대 위치 임베딩을 대체하며 더 우수. |
| **Cyclic shift + mask** | shift로 늘어난 윈도우를, 좌상단 순환 이동 + 마스킹으로 **정규 분할과 같은 개수**로 유지하는 효율 트릭. |
| **W→SW 두 블록 쌍** | W-MSA 블록과 SW-MSA 블록이 **한 쌍**으로 동작(논문 Eq. 3). 블록 수가 항상 짝수인 이유. |
